UML Diagrams

Intro to UML Diagrams

UML (Unified Modeling Language) class diagrams are one of the most commonly used diagrams in software engineering. They provide a visual representation of the static structure of a system, showing its classes, attributes, operations, and the relationships among objects. Class diagrams help in understanding and designing the system by illustrating the building blocks and their interactions.

Components of a Class Diagram
Classes:


Classes represent the blueprints of objects. Each class is depicted as a rectangle divided into three compartments:

- Top Compartment: Contains the class name, typically centered and bold.
- Middle Compartment: Lists the class attributes (fields or properties).
- Bottom Compartment: Lists the class methods (operations or functions).

Attributes:

Attributes are the data members of a class. They are represented with their visibility, name, and type.

Attributes:

Name: The name of the attribute.
Type: The data type of the attribute.
Example: -name: String indicates a private attribute name of type String.

Methods:

Methods are the functions or operations that a class can perform. They are represented with their visibility, name, parameter list, and return type.
Example: +getName(): String indicates a public method getName that returns a String.

Relationships:

Relationships in class diagrams show how classes are connected to each other. The primary types of relationships include:

- Association: A basic relationship where one class uses or interacts with another. Represented by a line connecting two classes, possibly with a label or arrow indicating direction.

- Multiplicity: Indicates the number of instances in a relationship (e.g., 1, 0..*, 1..*).

- Inheritance (Generalization): Indicates that one class (subclass) inherits from another (superclass). Represented by a solid line with a hollow triangle pointing towards the superclass.

- Aggregation: Represents a "whole-part" relationship where one class is a part of another. Depicted by a hollow diamond at the end of the association pointing to the whole.

- Composition: A stronger form of aggregation indicating that the part cannot exist without the whole. Represented by a filled diamond at the end of the association pointing to the whole.

- Dependency: Indicates that one class depends on another (uses it temporarily). Represented by a dashed line with an arrow pointing towards the class it depends on.
