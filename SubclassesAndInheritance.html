Subclasses and Inheritance

Intro to Subclasses and Inheritance

Inheritance is a fundamental principle of object-oriented programming in Java that allows a class (subclass or derived class) to inherit fields and methods from another class (superclass or base class). This mechanism promotes code reuse and establishes a hierarchical relationship between classes, where a subclass extends the functionality of a superclass.

Defining Subclasses

In Java, a subclass is created using the extends keyword. By extending a superclass, the subclass inherits its fields and methods, gaining all the non-private members of the superclass. This allows the subclass to reuse and build upon the existing functionality of the superclass.

Inheritance and Method Overriding

One of the key aspects of inheritance is method overriding. Method overriding allows a subclass to provide a specific implementation of a method that is already defined in its superclass. This enables polymorphism, where the same method can behave differently depending on the object that calls it.
When a method in a subclass has the same name, return type, and parameters as a method in its superclass, the subclass method overrides the superclass method. The @Override annotation is used to indicate that a method is overriding a superclass method, enhancing code readability and reducing errors.

Access Control and Inheritance

The access modifiers (private, protected, and public) control the visibility of fields and methods to subclasses:

- private members are not accessible in subclasses.
- protected members are accessible in subclasses and within the same package.
- public members are accessible from anywhere.
- Understanding these access levels is crucial for designing secure and maintainable class hierarchies.
